# V0.8 Feed 轻聚合方案决策

**状态（2026-05）**：**方案 B 已实现**（`cluster_id` + 列表聚合）；方案 A 仍适用于未入簇的单条信号。

## 方案 A（当前 — V0.8）

- **1 条 `feed_items` = 1 张首页卡片**
- 叙事化靠：`feed_type` 徽章、`narrative_hook` + `ai_summary`、主 Narrative Tag、`narrative_importance` 排序
- **不**引入 `cluster_id`、不合并多 URL 为单卡

## 方案 B（已登记 — 待后续实现）

> 产品已确认：先 A，后改 B。实现 B 时以本文件 + `CryptoPilot_V0.8_Narrative_Feed.md` §1.3 为准。

- 2–5 条相似 `feed_item` **合并展示为 1 卡**
- `related_source_count` **必须 ≥ 2**
- **仍不要** `market_insights` 主表
- 可在 `feed_items` 增加可选字段 **`cluster_id`**（同簇多条共享），列表 API 按簇聚合返回
- 首页卡片主文案仍为叙事 hook + summary，但来源数来自簇内多条

### 迁移到 B 时的检查清单

- [x] Prisma：`cluster_id`（可空索引）+ 种子/refresh 写入簇
- [x] `GET /api/feed?entity=feed_item`：按 `cluster_id` 聚合；单条 detail 路由保留
- [x] `FeedCard` / `related_source_count` / `related_sources` 与详情来源列表对齐簇
- [x] Admin：簇列表、代表条、解散簇、移出成员、重新聚类（`/admin/feed-clusters`）
- [x] 首页「信号流（聚合）」Tab + 叙事页相关 Feed 使用簇卡片
